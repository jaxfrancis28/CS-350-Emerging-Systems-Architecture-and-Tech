# CS-350-Emerging-Systems-Architecture-and-Tech

1. Summarize the project and what problem it was solving.
   
To enter the lucrative, multi-billion-dollar smart thermostat market, SysTec is developing a connected thermostat. The initial phase involves building a functional embedded prototype that manages low-level climate control (temperature sensing and HVAC adjustments) before introducing remote capabilities and data transmission to your analytics software over Wi-F.

My objectives were

•	Prototype Development: Create a low-level embedded system to handle primary thermostat functionalities.
•	Temperature Management: Accurately sense ambient temperature and toggle heating/cooling mechanisms to maintain user thresholds.
•	IoT Integration: Eventually expand the prototype to transmit telemetry and usage data over Wi-Fi to SysTec’s existing server analytics software.
In conclusion, SysTec’s smart thermostat prototype solves the problem of high, unregulated residential and commercial energy consumption by modernizing climate control.

5.	What skills from this project will be particularly transferable to other projects and/or coursework?
   
The skills from the project that will be particularly transferable to other projects and/or coursework are:

•	Modular Design: Breaking complex, monolithic problems into smaller, manageable, and testable components.
•	Memory Management: Understanding pointers, memory allocation, and resource handling (crucial for languages like C and C++).
•	Systems-Level Debugging: Utilizing tools to trace memory leaks, analyze process flow, and troubleshoot hardware-software interactions.
•	Version Control & Build Automation: Utilizing systems like Git and compiler toolchains.
